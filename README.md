### 初期設定の必要がないコミュニティプラグイン
- AutoLinkTitle
	- URLをペーストした際に、ページのタイトルを自動的に取得し、Markdownリンク (`[タイトル](URL)`) の形式で挿入してくれるプラグインです。手動でタイトルをコピー＆ペーストする手間を省けます。
- Calender
	- サイドバーやメインウィンドウにカレンダーを表示するプラグインです。クリックすることで特定の日付のデイリーノートを簡単に開いたり、新規作成したりできます。`obsidian-memos(Thino)` と連携し、カレンダー上にメモの存在を示すドットなどを表示することも可能です。
- DataView
	- ノート内のメタデータ（YAMLフロントマターやインラインフィールド形式 `Key:: Value`）を利用して、動的なリスト、テーブル、ビューを生成する非常に強力なプラグインです。SQLライクなクエリ言語を使い、特定のタグがついたノートの一覧や、特定の条件を満たすタスクリストなど、柔軟な情報収集と表示が可能です。
- ExcaliBrain
	- ノート間のリンク、タグ、フォルダ構造を元に、インタラクティブなナレッジグラフを生成するプラグインです。ノート同士の関連性を視覚的に把握し、アイデアの繋がりを発見するのに役立ちます。Zettelkastenのようなノート間の繋がりを重視する運用と特に相性が良いです。
- Excalidraw
	- Obsidian内で直接、手書きの図やスケッチを作成し、ノートに埋め込むことができるプラグインです。思考の整理、アイデアのブレインストーミング、複雑な概念の図解などに活用できます。作成した図はMarkdownファイルとして保存されます。
- recent-files-obsidian
	- 最近開いたファイルの一覧をサイドバーなどに表示するシンプルなプラグインです。作業中のファイルに素早くアクセスできるようになります。
- tag-wrangler
	- タグの一括リネーム、マージ、削除など、タグ管理を効率化するためのプラグインです。タグの整理やメンテナンスに役立ちます。
- templater-obsidian
	- 高機能なテンプレートエンジンで、新しいノートを作成する際に、あらかじめ定義したテンプレートを元に内容を自動挿入できます。日付、時刻、ユーザー定義の関数などをテンプレートに含めることができ、ノート作成の効率を大幅に向上させます。
- obsidian-memos(Thino)
	- X (旧Twitter) のようなインターフェースで、短いメモ（Memos）を素早く記録・管理できるプラグインです。デイリーノートとは別に、日常の気づきやアイデアを気軽に投稿し、後から見返したり、他のノートとリンクさせたりすることができます。記録されたメモは通常、指定したフォルダ（例: `Memos` フォルダ）以下に日付ごとのファイルとして保存されます。

### 初期設定が必要なコミュニティプラグイン

- [obsidian-git](https://github.com/Vinzent03/obsidian-git)
	- Obsidian VaultをGitリポジトリとして管理し、GitHubなどのリモートリポジトリと同期するためのプラグインです。これにより、複数デバイス間での設定やノートの同期、バージョン管理が可能になります。
	- **導入と設定のポイント**:
		- **前提**: ローカル環境にGitがインストールされている必要があります。基本的なGitの操作（コミット、プッシュ、プルなど）の知識があるとスムーズです。
		- **リポジトリの準備**: 既存のObsidian VaultをGitリポジトリとして初期化するか、既存のリモートリポジトリをクローンします。このテンプレートを利用する場合、プロジェクト内の`.git`フォルダを一旦削除し、ご自身の新しいリモートリポジトリに接続し直す必要があります。
		- **プラグイン設定**: コミット・プッシュ・プルの自動実行間隔、コミットメッセージのテンプレートなどを設定できます。モバイル環境での利用を考慮し、GitHubのアクセストークン（Personal Access Token）を利用した認証を推奨します。
		- **モバイル同期**: モバイル版Obsidianとも同期が可能です。設定方法はやや煩雑なため、専門の解説記事を参照してください。
		- **注意点**: 同期時のコンフリクト（競合）が発生する可能性があります。定期的なコミットとプッシュ、プルを心がけ、コンフリクト発生時は手動で解消する必要があります。
	- モバイルのObsidianアプリと課金せずに同期したいなら必要です。
	- 必須ではありませんが、やっておくと色々と便利です。

- [obsidian-kindle-plugin](https://github.com/hadynz/obsidian-kindle-plugin)
	- Kindleで作成したハイライトやメモをObsidianに同期するためのプラグインです。読書記録や引用の管理が効率化されます。
	- **導入と設定のポイント**:
		- **Amazonアカウント認証**: プラグイン設定画面から、お使いのAmazonアカウントのリージョン（例: `amazon.co.jp`）を選択し、再起動後、ログイン情報を入力して認証します。
		- **同期オプション**: 同期する書籍の選択、ハイライトの出力先フォルダやファイル名のカスタマイズ、ハイライトごとの日付フォーマット、書籍ごとのノート作成方法（1冊1ファイルか、全ハイライトを1ファイルにまとめるかなど）を設定できます。YAMLフロントマターに書籍情報（著者、ASINなど）を自動で追加するテンプレート機能もあります。
		- **同期の実行**: 手動で同期を実行するほか、Obsidian起動時に自動で同期するオプションも用意されています。
- [s3-image-uploader](https://github.com/jvsteiner/s3-image-uploader)
	- Obsidianに画像を貼り付けたりドラッグ＆ドロップした際に、自動的に指定したAWS S3バケットに画像をアップロードし、Markdownの画像リンクをS3上のURLに置き換えてくれるプラグインです。これにより、Vault自体のサイズを抑えつつ、画像をクラウドで管理できます。
	- **導入と設定のポイント**:
		- **AWSアカウントとS3の準備**: AWSアカウントが必要です。S3バケットを画像保存用に作成し、プログラムからのアクセスを許可するIAMユーザーを作成します。このIAMユーザーには、対象S3バケットへの書き込み権限（`s3:PutObject`, `s3:PutObjectAcl`など）を持つポリシーをアタッチします。
		- **プラグイン設定**: プラグイン設定画面で以下の情報を入力します。
			- `AWS Access Key ID` と `AWS Secret Key`: 作成したIAMユーザーのアクセスキーとシークレットキー。
			- `S3 Bucket Name`: 作成したS3バケット名。
			- `S3 Bucket Region`: S3バケットのリージョン (例: `ap-northeast-1`)。
			- `Base URL` (オプション): カスタムドメインを使用している場合などに設定。
			- その他、アップロード時のファイル名の命名規則や、画像の品質などを設定できます。
		- **動作**: 設定完了後、Obsidianエディタに画像をペーストするかドラッグ＆ドロップすると、自動的にS3にアップロードされ、ノート内のローカル画像パスがS3のURLに置き換えられます。
		- **コスト**: AWS S3には無料利用枠がありますが、保存容量やリクエスト数に応じて料金が発生します。個人利用であれば無料枠で収まることも多いですが、利用状況には注意が必要です。
- [LINE Notes Sync](https://note.com/shotovim/n/n55c363144d86)
	- LINEアプリからテキストメッセージや画像をObsidianに送信し、ノートとして保存できるプラグインです。外出先でのメモなどを手軽にObsidianに集約できます。
	- **導入と設定のポイント** (詳細はリンク先の記事を参照してください):
- [Obsidian\_to\_Anki](https://github.com/ObsidianToAnki/Obsidian_to_Anki)
	- Obsidianで作成したノートからAnkiカードを生成し、Ankiに送信するためのプラグインです。知識の定着や暗記学習を効率化します。
	- **導入と設定のポイント**:
		- **Anki側の準備**: Ankiデスクトップアプリケーションに `AnkiConnect` アドオンをインストールし、Ankiを起動した状態にしておく必要があります。AnkiConnectは、外部アプリケーション (この場合はObsidian) がAnkiと通信するためのAPIを提供します。
		- **カード化する範囲の指定**: Obsidianのノート内で、どの部分をAnkiカードにするかを指定します。一般的な方法としては、
			- 特定の区切り文字（例: `START` と `END`）で囲む。
			- 特定のタグ（例: `#card`）をつける。
			- 特定の見出しレベル以下の内容をカード化する。
			- 各行を個別のカード（表面/裏面）とする。
			- ノート全体を1つのカードとする。
			- これらの指定方法はプラグインの設定やノート内の指示（例: `TARGET DECK: デッキ名`）でカスタマイズ可能です。
		- **カードのフォーマット**: プラグイン設定やノート内の指示で、ノートのどの部分をカードの表面 (Front) と裏面 (Back) に割り当てるか、どのAnkiのノートタイプを使用するか、どのデッキに追加するかなどを細かく設定できます。Markdown形式で記述した内容（太字、リスト、画像など）もAnkiカードに反映されます。
		- **同期**: ObsidianからAnkiへの同期は、コマンドパレットから実行するか、ノート内のボタン（設定により表示可能）をクリックして行います。基本的にはObsidianからAnkiへの一方向同期です。
		- **メディアファイル**: ノートに埋め込まれた画像などもAnkiカードに含めることができますが、Ankiのメディア同期の設定も確認しておくと良いでしょう。
	- Anki側の設定でもアドオンの追加が必要です。
	- Ankiのメリットはこちら [Obsidianと連携する暗記アプリ、本当にAnkiでいいのか検証してみた｜けぽ](https://note.com/kepoorz/n/n6f8f72da106d)

### Cursor(AIエディタ)の役割

- 用意された複数の記事を1つのファイルにまとめる
- 出力されたファイルをアップデートする
- ファイル同士を紐づける
- Kindleなどの書籍引用部分を補足する(理解度テストを作ったり)
- 記事のタグづけを一貫性を持って行う
- Ankiデッキを自動で作成
- Slidev用のスライドを自動で作成
